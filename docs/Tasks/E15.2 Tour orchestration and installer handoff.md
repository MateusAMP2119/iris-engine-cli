# E15.2 Tour orchestration and installer handoff

Epic: [[Iris Epics]], E15 Onboarding and Guided Tour
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

The tour itself: a plain step table driven by a sequencer that explains, prompts, and
executes each step as a fresh in-process child app behind the `runStep` seam (real
command, real renderError, exit category returned) — workspace step (cwd or offer
`./iris-quickstart-demo`), engine install, start -d, info, sample apply, pipeline run,
provenance finale, wrap-up cheat-sheet. Adaptive skip when the workspace daemon answers
/healthz; decline/EOF/SIGINT abort clean (exit 0, resume hint); a failing step surfaces
its own error and exit category; `--yes` runs unattended; `--host` refused (exit 2).
Ends with the install.sh handoff (`/dev/tty` Y/n prompt, `exec "$dest/iris" quickstart
</dev/tty`, IRIS_FORCE/no-terminal fallback lines), README updates, and the conformance
leg proving the whole tour against a real engine.
