# E15.1 Quickstart surface and gating

Epic: [[Iris Epics]], E15 Onboarding and Guided Tour
Spec: [[Iris Specification Inventory]] (source of truth; on conflict the spec wins)

## Goal

Land `iris quickstart` as the third root verb with its two gates (interactivity: stdin
and stdout both TTY and `--json` off; color: the ceremony rule via makePainter) and its
three renderings — the interactive tour shell, the plain numbered copy-paste guide
(byte-stable, executes nothing, exit 0, golden-pinned), and the `--json` step-list
envelope (one data envelope of ordered (id, explanation, argv), executes nothing).
Ship the embedded `hello_iris` sample (seven rainbow colors into `demo.colors`,
POSIX-sh writer over `IRIS_DB_URL`, no lane) under `internal/cli/quickstartdata/`,
validated through the real declare loaders, materialized write-if-absent and never
clobbering. Includes the prerequisite `startDetached` refactor: rebuild the detach
child argv from the cobra command's visited flags instead of `os.Args`, so a future
in-process invocation of `engine start -d` can never re-exec the calling verb as the
daemon child (behavior-preserving; covered by the existing detach conformance test).
This task carries the full spec §8 delta (three-root-verbs amendment + four new Q/As),
all 14 E15 contract rows, and the `spec/inventory.lock` re-record.

## Depends on

E01 (tree, exit codes, painter), E03 (declare loaders); spec §8 delta rides this task.
